# MEMORY.md

本檔案記錄專案的關鍵決策、背景脈絡與重要事實，供 AI 代理與未來的自己快速理解「為什麼這樣做」。
與 `AGENTS.md`（告訴代理「怎麼做」）互補。

## 專案緣起

使用者希望建構一個「澳門註冊心理治療師執業地址分布地圖 APP」，讓市民能用地圖查找心理治療資源。

## 關鍵決策與理由

### 1. 平台：純靜態網頁（非原生 App）

- **決策**：Vite + 原生 JS + 靜態 JSON，部署到 GitHub Pages/Vercel。
- **理由**：資料量小（~90 人）、更新頻率低（半年一次）、公益性質。靜態網站零伺服器成本、易分享、SEO 友好、無需上架審核。
- **否決方案**：跨平台 App（React Native/Flutter）過重；前後端分離對 MVP 殺雞用牛刀。

### 2. 地圖服務：高德（非 Google/OSM）

- **決策**：高德地圖 JS API 2.0。
- **理由**：澳門街道細節完整、Web JS API 免費穩定、文件清楚、面向中文使用者體驗好。
- **考量**：Leaflet+OSM 雖免費但使用者偏好高德；Google Maps 有額度收費。

### 3. 資料來源與採集方式：官方衛生局直接爬取（突破 Cloudflare 驗證）

- **最終決策**：從**澳門特別行政區政府衛生局 — 從事私人醫務活動專業人員名冊**官網頁面全自動直接採集（`scripts/scrape.py`）。
- **理由與突破演變**：
  1. 使用者強烈要求以「官方衛生局資訊為準」。
  2. 衛生局系統設有 Cloudflare Turnstile 防爬蟲限制。
  3. **技術突破**：透過 Playwright 啟動 stealth 模式參數（傳入 `--disable-blink-features=AutomationControlled` 並過濾 `--enable-automation` 參數以隱藏 `navigator.webdriver` 自動化屬性），結合 headed 模式與 45s 超時，成功讓 Cloudflare 自動信任放行。
  4. 實現全自動分頁抓取（6 頁，116 筆記錄），解析得到 90 位心理治療師、41 個地點與 108 個執業關聯。
- **透明度與版權聲明**：頁尾只顯示「來源：澳門衛生局」，連結直接導向衛生局官方網站首頁（`https://www.ssm.gov.mo/`），而不顯示多餘的名冊網頁或「官方核實」等可能引發官方背書歧義的字眼。

### 4. 地圖視角：以「地點」為主

- **決策**：marker = 執業地點（41 個），點擊展開該地點的所有治療師。
- **理由**：
  - 更符合「找附近的心理治療資源」使用場景
  - pin 數量更少（41 vs 90+），地圖更清爽
  - 避免多地點治療師造成 pin 重疊問題

### 5. 不做評分/評論

- **決策**：明確排除評分、評論、推薦功能。
- **理由**：醫療專業倫理敏感，主觀評價可能誤導求助者、影響專業判斷，且有誹謗風險。

### 6. 人工校驗座標是必要流程

- **決策**：採集流程含 `preview.html` 人工校驗環節。
- **理由**：高德 geocoding 對澳門部分街名解析可能偏差，自動座標不可全信。校驗頁讓人在地圖上核對並修正。

### 7. 行動裝置介面：上下分屏（Split-Screen）+ AI 浮動按鈕

- **決策**：手機版（≤768px）改為「地圖固定上半屏（預設 40vh）+ 列表常駐下半屏」的分屏佈局，中間有可拖曳的把手調整比例（25%~70%）。AI 助理改為右下角浮動按鈕 → 全螢幕覆蓋層。
- **理由**：核心痛點是「地圖與列表永遠在搶空間」，分屏從根本上解決 — 兩者永遠同時可見。AI 是次要功能，不需要常駐佔位。

### 8. 官方心理健康資訊網與快測整合
- **決策**：首頁側邊欄頂部整合「官方心理健康服務」快捷卡片，手機端集成頂部快捷條。
- **理由**：將市民導向澳門衛生局官方**「自我心理狀態快測」**（`https://www.ssm.gov.mo/portal1/mentalhealth/kzJY9ECgaLx4vv83tK3eA?lang=ch`）以及**明愛 24 小時生命熱線 (2852 5222)**，滿足多級聯防要求，提高公共服務實用度。

### 9. 嚴格去 Slop 與 Emoji 禁令
- **決策**：應使用者要求，首頁 UI 快捷卡片、地點詳情面板、頁尾及 AI 助理回覆中**絕對禁止使用任何表情符號 (Emoji)**。
- **理由**：表情符號會破壞醫療資源導覽網頁的嚴肅性、專業度與極簡高級感（Anti-Slop）。

### 10. 個人隱私保護：中外文姓名僅擇一顯示
- **決策**：在地點詳情治療師列表中，**姓名僅展示中文姓名或英文姓名中的一個（優先中文名）**，不再並列顯示中英姓名對照，且 AI 助理回覆亦遵循此隱私原則。
- **理由**：減少非官方地圖中治療師個人完整真實姓名的多語曝光，保護個人隱私，但保留牌照號碼（License No.）展示以供核實。

### 11. 手機 App 喚醒與微信沙盒相容
- **決策**：行動端點擊導航優先調起原生 App（高德使用 `amapuri://`，Google 地圖在 iOS 使用 `comgooglemaps://`，Android 使用 `geo:`）。若 1.5 秒內未喚起則降級至 Web 瀏覽器版。特別是在微信內置瀏覽器中，會彈出高質感的磨砂 Dialog，引導使用者在外部瀏覽器開啟，或直接點擊按鈕跳轉至 H5 網頁版。
- **理由**：提供流暢的一鍵 App 導航體驗，並解決微信沙盒對自定義 URL Schemes 屏蔽的痛點。

### 12. v2.1 便民與安全升級（2026-07-06）
- **時段篩選走結構化解析而非 AI**：`hours.js` 把診時文字解析為可計算時段，「現在營業/週末/夜間」做成一鍵篩選按鈕。理由：最常用的查詢不該有 AI 延遲與不確定性；AI 透過 `find_open_locations` 工具共用同一套解析，答案與 UI 一致。
- **薄代理鎖定**：發現代理原樣轉發 `req.body`，任何人可直接 POST 指定貴的模型把網站當免費 LLM API。修復：`lib/copilot-proxy.js` 統一淨化 — model/max_tokens/temperature 伺服器強制、tools 白名單，dev 與正式共用同一邏輯。
- **金鑰洩漏修復**：`update_data.yml` 曾有硬編碼的高德 key fallback（且與前端 JS key 同一把）。已移除 fallback 並改為缺 secret 即報錯。**該 key 已進過版控歷史，需人工到高德後台作廢換新**。
- **深連結 + 分享**：`#loc=<id>` 直達機構，詳情面板「分享連結」。理由：心理健康資源的真實傳播路徑是社工/老師/親友轉介，可分享的連結比任何站內功能更便民。
- **附近優先**：Geolocation（WGS-84）→ `wgs84ToGcj02` 轉換後與高德座標比距離。澳門地區不轉換會偏數十至數百米。
- **分析隱私**：Vercel Analytics 停止上報搜尋詞內容，只記 `search_used` 行為 — 心理健康網站搜尋詞屬敏感資料。
- **資料漂移守衛**：`validate.py --baseline` 若治療師/地點數驟減 >30% 即失敗，防止官網改版導致每週自動更新靜默寫入殘缺資料；workflow 失敗自動開 issue 通知。
- **AI prompt 瘦身**：system prompt 的地點清單從全欄位 JSON 縮為 `id|名稱|分類` 索引，電話/診時/地址改由工具按需查詢，大幅降低每輪 token 消耗。
- **PWA**：manifest + Service Worker（stale-while-revalidate），資料僅數百 KB，離線可用成本極低。

## 實測中發現並修復的關鍵 bug

1. **微信內置瀏覽器屏蔽 Scheme 跳轉，引起導航按鈕卡死**：
   - 現象：微信內點擊「高德導航」或「Google 地圖」毫無反應。
   - 根因：微信屏蔽了所有外部 APP Schemes。
   - 修復：`detail.js` 增加 `MicroMessenger` 檢測，微信內直接彈出磨砂引導 Dialog。

2. **地圖詳情面板轉介引導框干擾與囉唆問題**：
   - 現象：詳情抽屜中塞入過多政府四級服務流程文字。
   - 根因：干擾了使用者查找治療師的核心體驗。
   - 修復：已依使用者指示完全移除四級聯防指引框，保持地圖詳情的清爽與簡潔。

3. **地圖初始化失敗**（`Map container div not exist`）：
   - 根因：高德 JS SDK 偏好以「容器 id 字串」初始化，直接傳 DOM 元素會在內部偵測時失敗。
   - 修復：`map.js` 改用 `container.id` 傳遞，並加 `requestAnimationFrame` 確保版面就緒。

4. **data.json 未打包進 dist**：
   - 根因：Vite 只打包被 import 的資源，`data/data.json` 是 `fetch()` 動態載入的，不會自動複製。
   - 修復：`vite.config.js` 加 `closeBundle` hook，打包後 `cpSync('data','dist/data')`。

5. **Geocoding 命中率與人工座標微調**：
   - 修正：在 `build_data.py` 中對特殊無地址機構（如山頂醫院、利達新邨社工局）進行精準對應與手動坐標修正，確保 41 個地點 100% 準確解析且全落在澳門陸地上。
