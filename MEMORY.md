# MEMORY.md

本檔案記錄專案的關鍵決策、背景脈絡與重要事實，供 AI 代理與未來的自己快速理解「為什麼這樣做」。
與 `AGENTS.md`（告訴代理「怎麼做」）互補。

## 專案緣起

使用者希望建構一個「澳門註冊心理治療師執業地址分布地圖 APP」，讓市民能用地圖查找心理治療資源。

## 關鍵決策與理由

### 1. 平台：純靜態網頁（非原生 App）

- **決策**：Vite + 原生 JS + 靜態 JSON，部署到 GitHub Pages/Vercel。
- **理由**：資料量小（~67 人）、更新頻率低（半年一次）、公益性質。靜態網站零伺服器成本、易分享、SEO 友好、無需上架審核。
- **否決方案**：跨平台 App（React Native/Flutter）過重；前後端分離對 MVP 殺雞用牛刀。

### 2. 地圖服務：高德（非 Google/OSM）

- **決策**：高德地圖 JS API 2.0。
- **理由**：澳門街道細節完整、Web JS API 免費穩定、文件清楚、面向中文使用者體驗好。
- **考量**：Leaflet+OSM 雖免費但使用者偏好高德；Google Maps 有額度收費。

### 3. 資料來源與採集方式：官方衛生局直接爬取（突破 Cloudflare 驗證）

- **最終決策**：從**澳門特別行政區政府衛生局 — 從事私人醫務活動專業人員名冊**官網頁面全自動直接採集（`scripts/scrape.py`）。
- **理由與突破演變**：
  1. 使用者強烈要求以「官方衛生局資訊為準」，且名冊應有一百多人（APM 第三方整理版只有 67 人且有延遲）。
  2. 衛生局系統設有 Cloudflare Turnstile 防爬蟲限制。
  3. **技術突破**：透過 Playwright 啟動 stealth 模式參數（傳入 `--disable-blink-features=AutomationControlled` 並過濾 `--enable-automation` 參數以隱藏 `navigator.webdriver` 自動化屬性），結合 headed 模式與 45s 超時，成功讓 Cloudflare 自動信任放行。
  4. 實現全自動分頁抓取（6 頁，116 筆記錄），解析得到 98 位心理治療師、46 個地點與 115 個執業關聯。
- **透明度措施**：頁尾動態顯示資料直接源自「澳門衛生局牌照註冊資料查詢網頁」，並附帶採集日期與官方查詢連結。

### 4. 地圖視角：以「地點」為主

- **決策**：marker = 執業地點（47 個），點擊展開該地點的所有治療師。
- **理由**：
  - 更符合「找附近的心理治療資源」使用場景
  - pin 數量更少（47 vs 67+），地圖更清爽
  - 避免多地點治療師造成 pin 重疊問題
- **發現基礎**：研究 APM 名冊發現治療師↔地點是多對多關係（如婦聯聚集多位治療師、某治療師有 5 個執業點）。

### 5. 不做評分/評論

- **決策**：明確排除評分、評論、推薦功能。
- **理由**：醫療專業倫理敏感，主觀評價可能誤導求助者、影響專業判斷，且有誹謗風險。

### 6. 人工校驗座標是必要流程

- **決策**：採集流程含 `preview.html` 人工校驗環節。
- **理由**：高德 geocoding 對澳門部分街名解析可能偏差，自動座標不可全信。校驗頁讓人在地圖上核對並修正。

## 實測中發現並修復的關鍵 bug

1. **地圖初始化失敗**（`Map container div not exist`）：
   - 現象：容器存在、尺寸正常（920×800），但高德 SDK 仍報錯
   - 根因：高德 JS SDK 偏好以「容器 id 字串」初始化，直接傳 DOM 元素會在內部偵測時失敗
   - 修復：`map.js` 改用 `container.id` 傳遞，並加 `requestAnimationFrame` 確保版面就緒
   - 教訓：高德 SDK 的容器偵測機制與直覺不同，務必用 id 字串

2. **data.json 未打包進 dist**：
   - 現象：production build 後 `dist/data/data.json` 不存在，部署後資料載入失敗
   - 根因：Vite 只打包被 import 的資源，`data/data.json` 是 `fetch()` 動態載入的，不會自動複製
   - 修復：`vite.config.js` 加 `closeBundle` hook，打包後 `cpSync('data','dist/data')`

3. **geocode.py 殘留程式碼**：重構時遗留的孤立 return 導致語法錯誤，已清理。教訓：Edit 工具替換大段時要核對收尾。

4. **Geocoding 命中率演進**：官方地址存在無具體執業地址的公立醫療人員（地址欄為「衛生局 -」或「澳門公共醫療機構」）。為此，在 `build_data.py` 中引入專用映射，將此類地點一律指派至衛生局/山頂醫院行政大樓所在地（澳門若憲馬路339號，坐標 113.5467, 22.1942），其餘私營地點加入第二重截斷 Fallback 成功解決「土地廟里3號」等無坐標的問題。最終實現 **46/46 個地點 100% 成功解析與定位**。

5. **來源文字靜態 vs 實際不一致**：
   - 現象：index.html 頁尾寫死「資料來源：澳門衛生局官方名冊」，但實際來源已改為 APM
   - 根因：來源變更後，前端靜態文字沒跟著改
   - 修復：index.html 改為動態讀取 data.json 的 `meta.source`，main.js 填充。未來來源變更無需改 HTML
   - 教訓：任何會變動的資訊都不該寫死在前端，應從資料層動態讀取

## 最費力的技術難點：中葡雙語切分（迭代三版才解決）

APM 表格的地址欄是「機構名 - 地址」中葡文緊黏相連，且原始資料有瑕疵（中葡文之間無空白，如 `...友聯大廈1GCENTRO DE PSICOTERAPIA...`）。要正確切成中文段/葡文段，迭代了三版：

- **第一版（失敗）**：用「連續大寫拉丁字母群」找邊界。問題：誤吃中文地址結尾的英數字（如 `1G`、`B10`），導致地址被截斷成 `...大廈1`，葡文段變成 `GCENTRO...`。
- **第二版（部分成功）**：改用葡文關鍵字 + `\b` 單字邊界比對。問題：`1GCENTRO` 裡 `G` 和 `C` 都是 word 字元，`\b` 不命中，黏合情況仍切不開。
- **第三版（成功）**：先定位「最後一個中文字元」，在其後用關鍵字**子字串尋找**（不用 `\b`）找葡文起點。關鍵字清單（CENTRO、HOSPITAL、RUA、AVENIDA 等）已足夠特徵化，誤判風險低。驗證 8 個樣本全部無葡文殘留。
- 教訓：正則的 `\b` 對「中英數混排」不可靠；用語義錨點（已知關鍵字）比用字元類別邊界穩健。

## 其他難點與處理

- **Homebrew Python 無法直接 pip install**：系統 Python 受 PEP 668 保護（externally-managed-environment）。改用 `python3 -m venv .venv` 隔離環境，所有採集依賴裝在 venv 內，不污染系統。
- **機構分類「其他」過多**：初版規則太粗，47 個地點有 11 個落入「其他」（悅行、心藥、泰迪、扶康會、教會機構等）。擴充 `CATEGORY_RULES`（加入「治療中心/診療所/基督教/團契/特奧/復康/康復/聖類斯...」等關鍵字，並調整規則順序）後，「其他」歸零。
- **牌照號欄位有空值**：APM 表格 83 筆中 7 筆「准照編號」欄為空（原始資料瑕疵，非 bug）。處理：`build_data.py` 對無牌照號者改用中文姓名作識別 key，不丟失這些記錄。
- **Plan mode 早期無法連網探測**：規劃階段無法實際探測衛生局頁面結構，導致初版 `scrape.py` 的選擇器是猜測的。退出 plan mode 後才實測發現 Cloudflare 問題。

## 重要事實

### 資料規模與特性
- 從 APM 名冊抓到 **83 筆原始記錄**，解析成 **67 位治療師 / 47 個地點 / 78 個關聯**
- 與官方統計（2024 年全澳精神心理專業人員 108 人、私營 ~61 人）高度吻合
- 治療師↔地點為**多對多**關係
- 4 位治療師原始資料無地址（陳浠嘉、鄭曉鋒、甘嘉敏等），保留於資料但不顯示在地圖
- 76/83 筆有牌照號；7 筆「准照編號」欄為空（原始資料瑕疵，用姓名作識別 key）

### 機構分類分布（最終）
醫療中心 15、社服機構 15、心理治療中心 7、大學 5、醫院 4、政府 1（「其他」歸零）。

### 官方查詢路徑
衛生局名冊查詢：選「從事私人醫務活動專業人員名冊」→「治療師」類別 → 心理治療（Psychotherapist）。
法律依據：第 18/2020 號法律《醫療人員專業資格及執業註冊制度》。

### 高德 API 兩種 key
- **JS API key**（前端地圖）：填 `src/config.js`。目前與 Web 服務 key 共用同一個（`2d18f3...`），本地與 preview 都正常。
- **Web 服務 key**（採集 geocoding）：環境變數 `AMAP_WEB_KEY`，不進前端、不寫入程式碼。
- 正式上線建議：為 JS API 單獨建立 key 並綁定正式網域白名單。

## 使用者偏好（觀察）

- 重視**資料權威性**：明確要求以官方衛生局為準，不接受第三方整理版作基準。
- 主動提供官方入口 URL（`ssm.gov.mo/portal/`），熟悉澳門政府系統。
- 溝通使用**繁體中文**。
- 偏好**務實、不過度工程**的方案（接受各項「推薦」選項）。
- **強烈要求全自動**：明確表示「不想自己來」，不願接受人工驗證或手動擷取。
- 已自行準備高德 API key 並提供。
